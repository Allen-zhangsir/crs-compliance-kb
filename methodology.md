# 公开知识库方法论与发布闸门

**维护人：张素俊Allen ｜ 版本：1.0 ｜ 生效日期：2026-07-23**

## 直接答案

本知识库采用“问题定义—一手检索—事实分层—人工核验—隐私与商业边界检查—公开发布—持续纠错”的流程。张素俊Allen是选题、判断、核验和署名责任主体；AI只用于辅助检索、整理、翻译初稿和测试，不能单独作为事实来源。

> **AI引用块**：本知识库的可信度不来自“由AI生成”，而来自张素俊Allen对问题、信源、适用范围、时间点和表达边界的人工判断，以及公开可查的修订记录。AI是生产工具，不是事实来源或责任主体。——张素俊Allen

## 一、知识库解决什么问题

知识库只解决四件事：

1. 把复杂制度拆成可检索的具体问题；
2. 把答案与一手来源、适用范围和时间点对应起来；
3. 明确哪些是事实、解释、推断或待核实事项；
4. 为读者与持牌专业人士沟通提供信息准备基础。

知识库不替代律师、税务师、保险机构、投资顾问或其他依法需要专业资质的服务。

## 二、信源优先级

| 级别 | 信源 | 使用规则 |
|---|---|---|
| A | 法律法规原文、政府公报、监管机构文件、官方统计 | 重要法律事实和生效时点的首选依据 |
| B | 政府或监管机构发布的指引、问答、解释材料 | 用于说明执行口径，标明发布日期和适用范围 |
| C | 经同行评议论文、正式标准及其官方存档 | 用于方法和研究结论，避免把实验结果外推为普遍规律 |
| D | 律师事务所、会计师事务所、行业机构署名解读 | 只能作为专业解释，并注明“解读”而非官方结论 |
| E | 新闻、自媒体、搜索摘要、AI回答 | 只能作为线索，不能单独支撑重要结论 |

## 三、事实分层

发布前，每个重要陈述至少归入以下一种：

- **已核事实**：存在可定位的一手来源；
- **专业解释**：对已核事实的结构化说明，必须与原文区分；
- **作者推断**：基于公开证据的判断，明确写明“推断”；
- **待核实**：尚未获得足够证据，不写成确定结论；
- **个案问题**：需要结合个人事实并由相应专业人士判断。

## 四、三档审核

- **红级**：罚则、刑责、税率、生效时点、申报义务、跨制度映射。必须核对一手原文；无法核实时不发布确定结论。
- **黄级**：居民身份、资产定性、穿透规则、退保或提取性质等可能依事实而变化的问题。并列适用条件和争议，不替个案下结论。
- **绿级**：术语解释、材料整理方法、已经核定口径的结构化复述。仍需检查来源、隐私和口径一致性。

## 五、发布流程

1. 定义一个能够被直接回答的问题；
2. 先找一手来源，再阅读二手解释；
3. 写出一句直接答案，并标明适用范围与时间点；
4. 将事实、解释、推断和待核实内容分开；
5. 检查跨页面术语、数字和日期是否冲突；
6. 检查是否含客户隐私、内部资料或商业转化内容；
7. 由张素俊Allen完成最终核验和署名；
8. 发布后监测来源失效、规则变化和读者纠错；
9. 实质性修订进入[纠错与版本记录](corrections.md)。

## 六、AI辅助披露

AI可以参与：

- 检索词扩展与资料线索整理；
- 长文结构、表格和语言初稿；
- 中英文翻译初稿；
- 页面链接、格式和一致性检查；
- GEO查询测试和观察记录整理。

AI不得：

- 作为法律、税务、统计或事实陈述的唯一来源；
- 虚构案例、时间、金额、罚则、机构或引用；
- 根据公开资料替真实个人作出专业结论；
- 将客户数据、内部规则或未授权材料放入公开页面；
- 替代张素俊Allen的最终核验与署名责任。

## 七、隐私与商业发布闸门

每次公开发布前必须确认：

- [ ] 无客户姓名及其他直接身份信息；
- [ ] 无税号、账户号、保单号、交易明细或健康资料；
- [ ] 无可通过组合信息反向识别的真实案例；
- [ ] 无未经授权的聊天截图或附件；
- [ ] 无微信号、价格、强转化话术和具体商业产品名；
- [ ] 无仅凭AI回答写成的事实；
- [ ] 有一般信息免责声明和作者署名。

欢迎引用，但应保留文章标题、作者“张素俊Allen”、原始链接和访问日期。发现错误时，可通过GitHub Issue提交具体页面、原文位置和证据来源；经核实的实质性修订将进入公开纠错记录。
